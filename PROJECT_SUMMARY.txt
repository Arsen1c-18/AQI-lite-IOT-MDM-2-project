## PAGES

### 1. LandingPage  (src/pages/LandingPage.jsx)
PURPOSE:
  Entry point of the app. Marketing/explainer page for the AQI Lite project.
SECTIONS:
  - Navbar: Logo (LeafLogo SVG) + nav links + CTA button "Open Dashboard"
  - Hero: Headline "Breathe smarter, live greener." + stat badges (4 sensors, 5min refresh, 24hr history, Open Source)
  - Mock Dashboard Preview: A visual replica of the real dashboard showing AQI 42, sensor cards
  - Floating Badges: Animated "Live Sync Active" and "ESP32 Online"
  - How It Works: Grid of 4 FeatureCards (ESP32 → Sensors → Supabase → React)
  - Sensor Strip: Describes PM2.5, CO2, Temp, Humidity using icon + description layout
  - CTA Section: Full-width green gradient card with "View Live Dashboard" button
  - Footer: Logo + links
DATA:
  None (static page)
DEPENDENCIES:
  framer-motion, lucide-react, react-router-dom, LeafLogo component

---

### 2. Dashboard  (src/pages/Dashboard.jsx)
PURPOSE:
  Main live monitoring view. Shows real-time AQI, sensor readings, device status, and recommendations.
SECTIONS:
  - AQIHero: Large hero card dynamically colored by AQI level (green=Good, yellow=Moderate, red=Unhealthy, etc.)
    Shows AQI number, category, improving/worsening trend compared to previous reading
  - SensorGrid: 4 cards for PM2.5 / CO2 / Temperature / Humidity with trend % badges
  - DeviceStatus: Shows device name, online/offline status (pulse dot), signal, power, last sync time
  - Recommendations (Health Action Plan): Shows 2 contextual cards based on AQI category
    "Open windows" OR "Close windows, turn on purifier" depending on air quality level
  - HistoricalChart: Area chart showing AQI + PM2.5 over last 24 hours
DATA:
  Real: Supabase tables aqi_results, sensor_readings, devices, device_status_logs  
  Fallback: Mock data (AQI=42, PM2.5=12.5, CO2=410, Temp=22.4, Humidity=48) used when DB is unreachable
HOOKS USED:
  useAQIData – fetches and auto-refreshes every 5 min, has fallback demo data
  useRealtime – Supabase postgres_changes subscription for live INSERT updates
DEPENDENCIES:
  All dashboard components, both hooks, framer-motion

---

### 3. HistoryPage  (src/pages/HistoryPage.jsx)
PURPOSE:
  Dedicated page for reviewing 24-hour historical data with filterable charts.
SECTIONS:
  - Header: Title + Metric Toggle Buttons (All Metrics / AQI Only / PM2.5 / CO2)
  - Stats Row: 4 stat cards – Current AQI, 24h Average, 24h Peak, 24h Best (min AQI)
  - Main Area Chart: AQI + PM2.5 as overlaid gradient area charts (Recharts AreaChart)
    Shows green gradient for AQI, red for PM2.5; has custom tooltip
  - CO2 Line Chart: Separate Recharts LineChart for CO2 trend (blue line), toggleable via filter
DATA:
  Same as Dashboard (useAQIData hook, falls back to generated 24-point mock data)
DEPENDENCIES:
  recharts (AreaChart, LineChart), useAQIData, framer-motion

## COMPONENTS

### Layout  (src/components/Layout.jsx)
  Wraps Dashboard, History, About pages (NOT LandingPage which has its own nav)
  - Sticky glassmorphism navbar with LeafLogo + "AQI Lite" brand
  - Active link detection using useLocation() from react-router-dom
  - Nav links: Home / Dashboard / History / About
  - Footer with logo, nav links

### LeafLogo  (src/components/LeafLogo.jsx)
  Inline SVG component. Two-leaf design with gradient (forest green to lime) and a dew drop.
  Props: className (default w-8 h-8)
  Used in: Layout, LandingPage (multiple sizes), AboutPage

### AQIHero  (src/components/AQIHero.jsx)
  Large dynamic hero card whose background color changes based on AQI category
  Color is pulled from getAqiInfo() in utils/constants.js → bgClass tailwind gradient
  Props: aqi, prevAqi

### SensorGrid  (src/components/SensorGrid.jsx)
  Grid of 4 glassmorphism sensor cards: PM2.5, CO2, Temperature, Humidity
  Each card shows value, unit, trend badge (up/down %)
  Props: data (object with pm25, co2, temperature, humidity)

### HistoricalChart  (src/components/HistoricalChart.jsx)
  Recharts AreaChart for AQI + PM2.5 over time
  Custom tooltip, gradient fills, reference line at AQI=100 (Moderate threshold)
  Props: data (array of { timestamp, calculated_aqi, pm25, co2 })

### DeviceStatus  (src/components/DeviceStatus.jsx)
  Shows device online/offline status, signal strength, power status, last sync time
  isOnline = last_seen within 10 minutes
  Props: deviceName, lastSeen, isOnline

### Recommendations  (src/components/Recommendations.jsx)
  Health Action Plan cards based on current AQI level
  Shows 2 contextual tips per level: "Safe to exercise" / "Turn on purifier" / "Stay indoors"
  Props: aqi

## HOOKS

### useAQIData  (src/hooks/useAQIData.js)
  Fetches latest AQI + sensor reading, historical 24h data, device info from Supabase
  On error: falls back to realistic mock data so UI always renders
  Auto-refreshes every 5 minutes as a backup to realtime
  Returns: { latestData, historicalData, deviceInfo, loading, error, refetch }

### useRealtime  (src/hooks/useRealtime.js)
  Creates a Supabase postgres_changes subscription on aqi_results table
  On new INSERT for matching DEVICE_ID: calls the onUpdate callback (triggers refetch)

## UTILITIES

### utils/constants.js
  AQI_LEVELS array: each level has min, max, category, bgClass (Tailwind gradient), textInfo, advice
  getAqiInfo(aqi) – returns matching level object
  DEVICE_ID – reads from VITE_DEVICE_ID env var, falls back to placeholder string

### supabaseClient.js
  Creates and exports Supabase client using VITE_SUPABASE_URL + VITE_SUPABASE_ANON_KEY

AQI Colors:
  Good (0-50):              green-500 → emerald-600
  Moderate (51-100):        yellow-400 → amber-500
  Sensitive (101-150):      orange-400 → orange-600
  Unhealthy (151-200):      red-500 → red-700
  Very Unhealthy (201-300): purple-500 → purple-800
  Hazardous (301+):         rose-800 → rose-950

## ENV VARIABLES REQUIRED
  VITE_SUPABASE_URL     – Supabase project URL
  VITE_SUPABASE_ANON_KEY – Supabase anon/public API key
  VITE_DEVICE_ID        – UUID of the registered ESP32 device in Supabase devices table
